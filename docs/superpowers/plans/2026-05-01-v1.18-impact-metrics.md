# v1.18+ Plan — Impact Metrics Surface

> **Date:** 2026-05-01
> **Status:** Active iteration (replaces v1.1-iteration.md as the current scope)
> **Live:** https://yingwen.io/evokernel-spec/

## Why this iteration

By v1.17 the project has saturated the "content density" axis: 39 hardware cards, 14 super-pods, 27 cases, 22 playbooks, 21 patterns, 16 fused-kernels, 19 models, 7-stage pipeline, /quality dashboard, /learn/deployment-failures/ guide, bidirectional model↔hardware recommender, coverage matrix, etc.

The *next* gap is not more content — it's **showing that the content matters**. A reader landing on the homepage today sees lots of data but no signal of who's using it, who else trusts it, or how it's growing. Without that, contributions stagnate and adoption flat-lines.

v1.18 closes this gap by adding an **impact-metrics surface**: a dashboard that quantifies adoption and a homepage strip that surfaces credibility signals.

## Scope (v1.18)

Five sub-features, each independently shippable:

### A. GitHub stars button (client-side)

**Why:** GitHub stars are the universal "useful" signal in the OSS ecosystem. Embedding a live count + one-click star on every page lifts conversion vs. an external link.

**Files:**
- `apps/web/src/components/impact/GitHubStarButton.tsx` — React island, fetches `https://api.github.com/repos/ying-wen/evokernel-spec` (no auth, public endpoint, 60 req/hour/IP unauthenticated). Caches in `localStorage` for 1 hour to amortize across pageviews. Falls back to badge if API fails.
- `apps/web/src/components/impact/GitHubStarButton.css`
- Mounted in `BaseLayout.astro` header right side.

**UX:** Shows `★ 1,234` count + "Star" CTA. Click opens GitHub auth flow → stars repo → returns. (Cannot one-click without OAuth; opening the GH star UI is the next-best.)

### B. Privacy-friendly visitor analytics

**Why:** Knowing which pages get traffic prioritizes next-round data work (the v1.18+ → v1.19+ chain). Privacy-friendly = no cookies, no PII, GDPR-clean.

**Approach:** Plausible Cloud free trial → eventually self-host on Cloudflare Pages Functions or deploy own Plausible instance. For v1.18, use a free tier OR a simple homegrown counter.

**Files:**
- `apps/web/src/layouts/BaseLayout.astro` — inject Plausible script tag via `<script defer data-domain="yingwen.io/evokernel-spec" src="https://plausible.io/js/script.js">`. Skip in dev/preview.
- Decision needed: Plausible (paid after trial) vs. self-hosted Umami vs. Cloudflare Web Analytics (free, less detail). **Default: Cloudflare Web Analytics** since GitHub Pages is fronted by yingwen.io which can be Cloudflare-proxied.

### C. /impact/ public dashboard page

**Why:** A single URL that summarizes "the project's heartbeat" — what new contributors and citers can link to.

**Files:**
- `apps/web/src/pages/impact.astro` — server-rendered page that pulls live numbers via React island.
- `apps/web/src/components/impact/ImpactDashboard.tsx` — fetches + displays:
  - GitHub stars / forks / watchers
  - Total pageviews (Cloudflare Analytics API or Plausible API)
  - Total content count (vendors / hardware / servers / cases / playbooks / patterns / fused-kernels) — derived from build manifest
  - Last commit date + last contributor (from git log baked at build)
  - Contributor count (`git shortlog -sn` baked at build)
  - Last 5 cases / playbooks added (from data file mtime)
  - Citations — manually curated `data/citations.yaml` (papers, blogs, conference talks that link here)

### D. Homepage impact strip

**Why:** First-page surface — under the hero, above the catalog grid. Reduces "is this active?" doubt for first-time visitors.

**Files:**
- `apps/web/src/components/impact/ImpactStrip.astro` — small horizontal band: `★ stars · 👥 contributors · 📦 N cases · 🚀 last commit Xd ago · ↗ /impact/`
- Wired into `apps/web/src/pages/index.astro` after hero.

### E. Citation tracker

**Why:** Citations are the strongest "this matters" proof. Static YAML keeps it under contributor control (PR-able).

**Files:**
- `data/citations.yaml` — array of `{ title, url, source_type, date, snippet }`
- `schemas/citation.ts` — schema entry
- `apps/web/src/pages/impact.astro` — render Citations section grouped by source_type (paper / blog / talk / docs)

## Acceptance criteria

1. `pnpm validate` ✓ (citation schema valid)
2. `pnpm build` ✓ (one extra page: /impact/)
3. Homepage shows live star count fetched from GitHub API
4. /impact/ page renders all 6 metric categories with actual numbers (stars, pageviews, content counts, last commit, contributors, citations)
5. Cloudflare Web Analytics or Plausible script injected sitewide (excluding dev)
6. New E2E tests pass (≥6 tests covering star button, impact strip, /impact/ page, citation listing)
7. CHANGELOG updated, README badges updated, tag v1.18.0 pushed
8. Production live at https://yingwen.io/evokernel-spec/impact/
