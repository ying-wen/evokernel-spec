# Contributors

> Everyone who has contributed code, data, or substantive review to
> EvoKernel Spec. Thank you. 🙇

## How to get on this list

- Open a PR (any size — typo fixes count) and your GitHub handle gets added on merge.
- Submit a deployment case via [the case template](https://github.com/ying-wen/evokernel-spec/issues/new?template=03-deployment-case.yaml).
- Submit a vendor data correction via [the vendor-claim template](https://github.com/ying-wen/evokernel-spec/issues/new?template=01-vendor-data-claim.yaml).
- Or contribute via any of the three [tracks on /contribute](https://github.com/ying-wen/evokernel-spec/blob/main/apps/web/src/pages/contribute.astro).

PR authors are auto-credited in the next tagged release notes
(via `git log <prev-tag>..<this-tag>` in `release.yml`). This file
is the curated lifetime roster.
